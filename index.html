HTML

What is HTML and why is it important?

HTML (HyperText Markup Language) is the standard markup language used to create web pages. It provides the structure and content of websites by using various elements and tags. HTML is important because it forms the foundation of web development, allowing browsers to interpret and display content correctly. Every website you visit uses HTML to organize text, images, links, and other media.

HTML

What is the difference between HTML elements and tags?

HTML tags are the markup syntax using angle brackets (e.g., <p>, <div>), while an HTML element consists of the opening tag, content, and closing tag together. For example, <p> is a tag, but <p>Hello World</p> is a complete paragraph element. Tags are the code, elements are the complete structure.

HTML

What are semantic HTML elements?

Semantic HTML elements clearly describe their meaning and purpose both to the browser and developer. Examples include <header>, <nav>, <article>, <section>, <aside>, and <footer>. These elements improve code readability, accessibility, and SEO compared to generic <div> elements. They help screen readers and search engines better understand the content structure.

HTML

How do you create a hyperlink in HTML?

You create hyperlinks using the <a> (anchor) tag with the href attribute. For example: <a href="https://example.com">Click here</a>. You can also use target="_blank" to open links in a new tab, and the href attribute can point to external URLs, internal pages, email addresses (mailto:), or page sections using IDs (#section).

CSS

What is CSS and how does it work with HTML?

CSS (Cascading Style Sheets) is a styling language used to control the presentation and layout of HTML elements. While HTML provides structure, CSS adds visual styling like colors, fonts, spacing, and positioning. CSS works by selecting HTML elements and applying style rules to them. It can be added inline, internally in a <style> tag, or externally in separate .css files.

CSS

What is the box model in CSS?

The CSS box model is a fundamental concept that describes how elements are rendered. Every element is a rectangular box consisting of four areas: content (the actual content), padding (space around content), border (surrounds the padding), and margin (space outside the border). Understanding the box model is crucial for controlling layout and spacing. Use box-sizing: border-box to include padding and border in the element's total width and height.

CSS

What is the difference between Flexbox and Grid?

Flexbox is a one-dimensional layout system designed for arranging items in rows or columns, ideal for component layouts and alignment. CSS Grid is a two-dimensional system that handles both rows and columns simultaneously, perfect for complex page layouts. Use Flexbox for navigation bars, card layouts, and centering. Use Grid for overall page structure, photo galleries, and complex multi-column designs.

CSS

How do CSS selectors work?

CSS selectors target HTML elements to apply styles. Common selectors include: element selectors (p, div), class selectors (.className), ID selectors (#idName), attribute selectors ([type="text"]), and pseudo-classes (:hover, :first-child). You can combine selectors for more specific targeting. Understanding selector specificity is important as it determines which styles are applied when multiple rules conflict.

CSS

What are CSS preprocessors like Sass and why use them?

CSS preprocessors like Sass, Less, and Stylus extend CSS with features like variables, nesting, mixins, functions, and imports. They compile to regular CSS. Benefits include: better organization with nesting, reusable code with mixins, mathematical operations, color functions, and modular code structure. Preprocessors make large stylesheets more maintainable, reduce repetition, and enable more powerful styling capabilities while keeping the output as standard CSS.

CSS

How do you make a website responsive?

Make websites responsive using: 1) Flexible grid layouts with percentages or CSS Grid/Flexbox. 2) Media queries to apply different styles at different screen sizes. 3) Responsive images using max-width: 100%. 4) Viewport meta tag for proper mobile scaling. 5) Relative units (em, rem, %, vw, vh) instead of fixed pixels. 6) Mobile-first approach. 7) Testing on actual devices. Modern CSS features like clamp(), min(), and max() also help create truly responsive designs.

JavaScript

What is JavaScript and what is it used for?

JavaScript is a high-level, interpreted programming language that adds interactivity and dynamic behavior to websites. It runs in the browser and can manipulate HTML and CSS, respond to user events, make HTTP requests, store data, and much more. JavaScript is essential for modern web development and is also used for server-side development (Node.js), mobile apps, desktop applications, and game development.

JavaScript

What is the difference between let, const, and var?

var is function-scoped, can be redeclared and updated, and is hoisted. let is block-scoped, cannot be redeclared but can be updated, and is also hoisted but not initialized. const is block-scoped, cannot be redeclared or reassigned, and must be initialized when declared. Modern JavaScript best practices recommend using const by default, let when you need to reassign, and avoiding var entirely.

JavaScript

What are arrow functions in JavaScript?

Arrow functions are a shorter syntax for writing functions introduced in ES6. They use the => syntax. For example: const add = (a, b) => a + b. Arrow functions have lexical 'this' binding, meaning they inherit 'this' from their parent scope. They're great for callbacks and short functions but cannot be used as constructors and don't have their own arguments object.

JavaScript

What is the DOM and how do you manipulate it?

The DOM (Document Object Model) is a programming interface that represents HTML documents as a tree structure of objects. JavaScript can manipulate the DOM to change content, structure, and styling dynamically. Common methods include getElementById(), querySelector(), createElement(), appendChild(), and manipulating properties like innerHTML, textContent, and classList. Understanding DOM manipulation is fundamental to creating interactive web pages.

JavaScript

What is event delegation in JavaScript?

Event delegation is a technique where you attach a single event listener to a parent element instead of multiple listeners to child elements. It leverages event bubbling to handle events on child elements. This improves performance, uses less memory, and works for dynamically added elements. For example, instead of adding click listeners to all list items, add one to the parent ul and check event.target to determine which item was clicked.

JavaScript

What are Promises and async/await in JavaScript?

Promises represent the eventual completion or failure of an asynchronous operation. They have three states: pending, fulfilled, or rejected. Promises use .then() for success and .catch() for errors. Async/await is syntactic sugar built on Promises, making asynchronous code look synchronous and easier to read. Use 'async' before a function and 'await' before Promises. Always wrap await calls in try/catch blocks for error handling.

Java

What is Java and where is it used?

Java is a high-level, object-oriented programming language known for its "write once, run anywhere" philosophy. It's platform-independent because Java code compiles to bytecode that runs on the Java Virtual Machine (JVM). Java is widely used for enterprise applications, Android app development, web applications, big data technologies, scientific applications, and backend systems. It's valued for its reliability, security, and scalability.

Java

What is the difference between JDK, JRE, and JVM?

JVM (Java Virtual Machine) executes Java bytecode and provides platform independence. JRE (Java Runtime Environment) includes the JVM plus libraries needed to run Java applications. JDK (Java Development Kit) includes the JRE plus development tools like compilers and debuggers needed to develop Java applications. Developers need JDK, while end-users only need JRE to run Java programs.

Java

What are the main principles of Object-Oriented Programming in Java?

The four main OOP principles are: 1) Encapsulation - bundling data and methods that operate on that data within classes, hiding internal details. 2) Inheritance - creating new classes from existing ones to promote code reuse. 3) Polymorphism - objects taking many forms, allowing methods to do different things based on the object. 4) Abstraction - hiding complex implementation details and showing only necessary features.

Java

What is the difference between String, StringBuilder, and StringBuffer?

String is immutable - once created, it cannot be changed. Every modification creates a new String object. StringBuilder is mutable and not thread-safe, making it faster for single-threaded scenarios. StringBuffer is also mutable but thread-safe (synchronized), making it slower but safe for multi-threaded environments. Use String for constant values, StringBuilder for single-threaded string manipulation, and StringBuffer for multi-threaded scenarios.

Android

What is Android and how does Android development work?

Android is an open-source mobile operating system based on Linux, developed by Google. Android development involves creating applications for Android devices using languages like Java or Kotlin. Development is typically done in Android Studio, Google's official IDE. Android apps consist of components like Activities, Services, Broadcast Receivers, and Content Providers, with UI designed using XML layouts or Jetpack Compose.

Android

What is an Activity in Android?

An Activity represents a single screen with a user interface in an Android app. It's a crucial component that users interact with. Activities have a lifecycle with methods like onCreate(), onStart(), onResume(), onPause(), onStop(), and onDestroy() that the system calls as the activity moves through different states. Understanding the activity lifecycle is essential for managing resources, saving state, and creating responsive apps.

Android

What is the difference between Intent and Intent Filter?

An Intent is a messaging object used to request an action from another component, like starting an activity or service. There are explicit intents (specifying the component) and implicit intents (declaring a general action). Intent Filters are declared in the manifest and specify what types of intents a component can receive. They define actions, categories, and data types that an activity, service, or broadcast receiver can handle.

Android

What is RecyclerView and why use it instead of ListView?

RecyclerView is an advanced and flexible version of ListView for displaying large datasets. It recycles views that are no longer visible, improving performance and memory usage. RecyclerView requires a ViewHolder pattern, supports different layout managers (Linear, Grid, Staggered), provides better animations, and allows more customization. It's the modern standard for displaying lists in Android and offers better performance than ListView.

Git & GitHub

What is Git and why is it important?

Git is a distributed version control system that tracks changes in source code during software development. It allows multiple developers to work together, maintains a complete history of changes, enables reverting to previous versions, and supports branching and merging. Git is essential for modern software development as it provides collaboration, backup, and the ability to experiment without affecting the main codebase.

Git & GitHub

What is the difference between Git and GitHub?

Git is the version control system software that runs locally on your computer, tracking changes in your code. GitHub is a cloud-based hosting service for Git repositories that provides a web interface, collaboration features, issue tracking, pull requests, and social coding features. Git works offline and is the tool itself, while GitHub is an online platform that makes it easier to collaborate and share Git repositories with others.

Git & GitHub

What are the basic Git commands every developer should know?

Essential Git commands include: git init (initialize a repository), git clone (copy a repository), git add (stage changes), git commit (save changes), git push (upload to remote), git pull (download from remote), git status (check status), git branch (manage branches), git checkout (switch branches), git merge (combine branches), and git log (view history). Mastering these commands is fundamental to using Git effectively.

Git & GitHub

What is a branch in Git and why use branching?

A branch is an independent line of development in Git that allows you to work on features or fixes without affecting the main codebase. The default branch is usually called 'main' or 'master'. Branching allows developers to experiment, develop features in isolation, collaborate without conflicts, and maintain different versions. Common practices include creating feature branches, development branches, and release branches to organize workflow effectively.

SQL

What is SQL and what is it used for?

SQL (Structured Query Language) is a standard language for managing and manipulating relational databases. It's used to create, read, update, and delete data (CRUD operations), define database structure, control access, and query data. SQL is essential for backend development, data analysis, and working with databases like MySQL, PostgreSQL, Oracle, and SQL Server. Learning SQL is crucial for any developer working with data-driven applications.

SQL

What is the difference between SQL and NoSQL databases?

SQL databases are relational, use structured schemas with tables and relationships, and use SQL for queries. They ensure ACID properties and are great for complex queries and transactions. NoSQL databases are non-relational, have flexible schemas, and can be document-based, key-value, graph, or column-family. They're designed for scalability and handling large volumes of unstructured data. Choose SQL for structured data with complex relationships, NoSQL for flexibility and horizontal scaling.

SQL

What are primary keys and foreign keys?

A primary key is a column or set of columns that uniquely identifies each row in a table. It must contain unique values and cannot be NULL. Each table can have only one primary key. A foreign key is a column that creates a relationship between two tables by referencing the primary key of another table. Foreign keys maintain referential integrity and enable database relationships, allowing you to connect related data across tables.

SQL

What are JOINs in SQL and what types exist?

JOINs combine rows from two or more tables based on related columns. Main types include: INNER JOIN (returns matching rows from both tables), LEFT JOIN (all rows from left table, matching from right), RIGHT JOIN (all rows from right table, matching from left), and FULL OUTER JOIN (all rows from both tables). Understanding JOINs is essential for working with relational databases and retrieving data from multiple related tables.

Interview

How should I prepare for a frontend developer interview?

Prepare by mastering HTML, CSS, and JavaScript fundamentals. Practice coding challenges on platforms like LeetCode or HackerRank. Build projects to showcase your skills. Study common interview topics: responsive design, browser compatibility, performance optimization, accessibility, and modern frameworks. Review data structures and algorithms. Prepare questions about the company. Practice explaining your thought process. Have a portfolio ready and be prepared to discuss your projects in detail.

Interview

What are common behavioral interview questions?

Common questions include: "Tell me about yourself", "Why do you want this job?", "Describe a challenging project", "How do you handle conflicts?", "What's your biggest weakness?", "Where do you see yourself in 5 years?", and "Tell me about a time you failed". Use the STAR method (Situation, Task, Action, Result) to structure answers. Be honest, specific, and focus on what you learned and how you grew from experiences.

Interview

What questions should I ask the interviewer?

Ask about: the team structure and who you'd work with, day-to-day responsibilities, technologies and tools used, code review process, professional development opportunities, company culture and values, current challenges the team faces, success metrics for the role, and next steps in the hiring process. Asking thoughtful questions shows genuine interest and helps you determine if the role is right for you.

Interview

How do I explain gaps in my resume or lack of experience?

Be honest and positive. Focus on what you did during gaps: learning, personal projects, freelancing, volunteering, or personal circumstances. Emphasize transferable skills and how you've grown. For lack of experience, highlight relevant projects, coursework, bootcamps, self-learning, and enthusiasm to learn. Demonstrate your passion through portfolio projects and knowledge of current technologies. Show how your unique background provides valuable perspective.
